## implementation highlights
Use diverse augmentations to improve model generalization and prevent overfitting. Including random cropping, horizontal and vertical flipping, rotation, color jittering, grayscale, and random erasing. Train for longer epochs while not overfitting.

## data preprocessing strategy
For Unet:
- Randomly crop the image to `512x512`, then pad it to `572x572` using `cv2.BORDER_REFLECT`, and finally center crop the mask to `388x388`.
For ResNet34_UNet:
- Directly resize/crop the input to `256x256` without UNet-style padding/cropping, and output the same size as input.
- Normalize with mean = (0.5, 0.5, 0.5) and std = (0.5, 0.5, 0.5)


## training strategy
for unet, use longer warmup and larger weight decay (1e-4) to stabilize training. For resnet34_unet, use shorter warmup and smaller weight decay (1e-5) to allow faster convergence.
gradient clipping is essential for unet to prevent loss spikes, while resnet34_unet is more stable and can be trained with a larger learning rate.

## key observations
- UNet lacks batch normalization and attention mechanisms, making it more sensitive to hyperparameters and unstable during training, with more loss spikes in the curve (requiring smaller learning rates and gradient clipping). ResNet34_UNet's ResNet34 backbone and CBAM attention enable the model to learn effective features more easily, resulting in more stable training.

## comparison between UNet and ResNet34_UNet
- Encoder part
    - Unet: only convolutional layers and ReLU, no batch normalization.
    - ResNet34_UNet: ResNet34 backbone with batch normalization and skip connections.
- Decoder part
    - Unet: upsampling with transposed convolution, no attention.
    - ResNet34_UNet: upsampling with bilinear interpolaration, with CBAM attention modules in each decoder block.
